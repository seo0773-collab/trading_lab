
# Codex 작업 요청서 수정안: DI Kalman M/W 패턴 기반 매매 전략 및 백테스트 구현

## 0. 수정 목적

기존 작업 요청서는 `+DI / -DI Kalman 극점 기반 M/W 패턴 연구`와 `P5 예측 통계 생성`에 초점이 있었다.

이번 수정 목적은 다음과 같다.

```text
패턴 연구용 리포트 생성
→ 실제 Long / Short / Hold 매매 신호 생성
→ 진입 / 익절 / 손절 / 청산 조건 정의
→ train / validation / test 백테스트 가능 구조 구현
```

TradingView와의 수치 비교는 이번 작업 범위에서 제외한다.
중요한 것은 Python 내부에서 일관된 계산과 백테스트가 가능한 구조를 만드는 것이다.

## 1. 최종 목표

`trading_lab` 구조에 맞춰 다음을 구현한다.

### 1차 목표

`+DI / -DI Kalman M/W 패턴`을 이용해 매매 신호를 생성한다.

신호 후보:

```text
Long setup:
+DI Kalman = W형
-DI Kalman = M형
+DI 최근 4극점 평균 > -DI 최근 4극점 평균
pressure_adjusted_expected_value > 0

Short setup:
+DI Kalman = M형
-DI Kalman = W형
-DI 최근 4극점 평균 > +DI 최근 4극점 평균
pressure_adjusted_expected_value > 0
```

### 2차 목표

두 가지 진입 방식을 모두 백테스트한다.

```text
Variant A: P4 Prediction Entry
4번째 극점 확정 시점에 P5를 예측하고 다음 봉에 진입한다.

Variant B: P5 Confirmation Entry
5번째 극점이 실제로 higher-low / lower-high 형태로 확인된 뒤 다음 봉에 진입한다.
```

### 3차 목표

익절 / 손절 / 청산 조건을 여러 방식으로 실험 가능하게 만든다.

```text
1. ATR Stop + R-multiple Take Profit
2. 최근 가격 swing high/low 기반 stop
3. opposite DI pattern 발생 시 청산
4. time stop
5. trailing stop
```

### 4차 목표

충분한 train 구간을 확보하고, walk-forward 또는 chronological split 기반으로 validation/test를 수행한다.

## 4. 충분한 Train 구간 조건

train 구간은 반드시 충분해야 한다.

권장 기준:

```text
crypto 4H 기준:
train 최소 3년 이상
또는 최소 5,000개 이상 candle
또는 최소 300개 이상 pattern event

crypto 1H 기준:
train 최소 2년 이상
또는 최소 10,000개 이상 candle
또는 최소 500개 이상 pattern event

daily/weekly 주식 기준:
가능하면 10년 이상
또는 최소 1,500개 이상 candle
```

데이터가 부족하면 해당 symbol/timeframe은 전략 평가에서 제외하거나 `insufficient_train_data`로 표시한다.

랜덤 split은 금지한다.

사용할 split 방식:

```text
1. Chronological split
   train 60%
   validation 20%
   test 20%

2. Walk-forward split
   train window 충분히 확보
   validation/test를 시간순으로 이동
```

예시:

```text
BTCUSDT 4H, 2020~2026 데이터 기준

train:
2020-01-01 ~ 2023-12-31

validation:
2024-01-01 ~ 2024-12-31

test:
2025-01-01 ~ 2026-현재
```

## 5A. 극점 추출 및 M/W 패턴 정의 (신규 보강)

이 섹션은 원본 작업 요청서에 정량 정의가 없던 부분을 구체화한 기본안이다.
아래 정의는 모두 config로 분리해 변경 가능하게 구현한다.

### 5A.1 극점 추출 (Kalman line 기준)

극점은 +DI Kalman, -DI Kalman 각각에 대해 독립적으로 추출한다.
극점은 고점/저점이 반드시 교대(alternating)로 나오도록 한다.

reversal threshold 정의:

```python
rev_threshold = reversal_mult * rolling_std(kalman_series, reversal_std_window)

# 기본값 후보
reversal_mult = 1.0
reversal_std_window = 50
```

상태 머신:

```text
1. 직전 확정 극점 이후 구간의 running max / running min을 추적한다.
2. 고점 확정: kalman 값이 running max에서 rev_threshold 이상 하락하면
   running max가 발생한 bar(idx)를 고점 극점으로 확정한다.
   이 확정이 일어난 bar가 confirmation_idx다.
3. 저점 확정: 반대 방향 동일 로직.
4. 극점 idx와 confirmation_idx는 반드시 분리해 저장한다.
   신호 생성과 백테스트는 항상 confirmation_idx 기준으로만 동작한다.
```

### 5A.2 M/W 패턴 정의 (4극점 확정 시점)

P1~P4는 가장 최근에 확정된 4개 극점이다 (P4가 최신).

```text
W형 (상승 전환 후보):
  극점 배열: P1(저) P2(고) P3(저) P4(고)
  필수 조건: P3 > P1            # higher low
  P5 예측: P5는 저점이며 P5 > P3 기대 (higher low)

M형 (하락 전환 후보):
  극점 배열: P1(고) P2(저) P3(고) P4(저)
  필수 조건: P3 < P1            # lower high
  P5 예측: P5는 고점이며 P5 < P3 기대 (lower high)
```

선택적 강화 조건 (config로 on/off, 기본 off):

```text
W형: P4 > P2 (higher high)
M형: P4 < P2 (lower low)
```

패턴 판정 시점은 P4의 confirmation_idx다.
P4 확정 이전 bar에서는 해당 패턴이 존재하지 않는 것으로 취급한다.

### 5A.3 P5 확인 정의 (Variant B용)

```text
P5 confirmation (long):
  +DI W 상태에서 다음 저점 극점이 확정되고 P5 > P3
  -DI M 상태에서 다음 고점 극점이 확정되고 P5 < P3

P5 confirmation (short):
  +DI M 상태에서 다음 고점 극점이 확정되고 P5 < P3
  -DI W 상태에서 다음 저점 극점이 확정되고 P5 > P3

진입 가능 시점 = P5의 confirmation_idx 다음 봉 open
```

### 5A.4 DMI 계산

DMI(+DI / -DI)는 저장소에 기존 구현이 없다.
Wilder smoothing 기반 표준 정의로 직접 구현하거나
`ta` 라이브러리(ADXIndicator의 adx_pos / adx_neg)를 사용한다.
기본값: di_len = 14. 어느 쪽을 쓰든 구현을 config에 기록해 재현 가능하게 한다.

## 6. DI Pressure 기반 기대값 보정

최근 4개 극점 평균을 이용해 방향성 압력 우위를 계산한다.

```python
eps = 1e-9

plus_extreme_mean_4 = mean(latest_4_plus_di_kalman_extreme_values)
minus_extreme_mean_4 = mean(latest_4_minus_di_kalman_extreme_values)

di_pressure_spread = (
    plus_extreme_mean_4 - minus_extreme_mean_4
) / max(plus_extreme_mean_4 + minus_extreme_mean_4, eps)

long_pressure_score = plus_extreme_mean_4 / max(
    plus_extreme_mean_4 + minus_extreme_mean_4,
    eps,
)

short_pressure_score = minus_extreme_mean_4 / max(
    plus_extreme_mean_4 + minus_extreme_mean_4,
    eps,
)

long_rr_factor = plus_extreme_mean_4 / max(minus_extreme_mean_4, eps)
short_rr_factor = minus_extreme_mean_4 / max(plus_extreme_mean_4, eps)

long_rr_factor = clip(long_rr_factor, 0.50, 2.00)
short_rr_factor = clip(short_rr_factor, 0.50, 2.00)

# 신규 보강: §5 feature의 pressure_score / pressure_rr_factor 정의
# 신호 방향에 해당하는 값을 그대로 사용한다.
if predicted_direction == "long":
    pressure_score = long_pressure_score
    pressure_rr_factor = long_rr_factor
else:
    pressure_score = short_pressure_score
    pressure_rr_factor = short_rr_factor

pressure_aligned = pressure_score > 0.5
```

해석:

```text
long_pressure_score > 0.5
→ +DI 압력이 우세
→ long setup의 손익비가 좋아질 가능성

short_pressure_score > 0.5
→ -DI 압력이 우세
→ short setup의 손익비가 좋아질 가능성
```

패턴 방향과 pressure 방향이 맞지 않으면 신호를 제거하지 말고, 우선 기대값을 감점한다.

## 7. 기대값 계산

기본 기대값:

```python
raw_expected_value = (
    win_probability * avg_win
    -
    (1.0 - win_probability) * avg_loss
    -
    costs
)
```

pressure 보정 기대값:

```python
pressure_adjusted_expected_value = (
    win_probability * avg_win * pressure_rr_factor
    -
    (1.0 - win_probability) * avg_loss / max(pressure_rr_factor, eps)
    -
    costs
)
```

초기 연구 단계에서는 `pressure_adjusted_expected_value > 0`인 경우만 trade candidate로 분류한다.

## 7A. win_probability / avg_win / avg_loss 추정 방법론 (신규 보강)

§7의 기대값 계산에는 win_probability, avg_win, avg_loss가 필요하다.
이 통계는 트레이드 결과에서 나오고, 트레이드는 EV 필터를 통과한 신호에서 나오므로
그대로 구현하면 순환 참조가 생긴다. 이를 2-Pass 방식으로 해결한다.

### Pass 1: train 구간 통계 생성

```text
1. train 구간에서 EV 필터 없이 모든 pattern event에 대해
   기준 청산 규칙(ATR 2.0 stop + fixed 2R TP + time stop 48 bars)으로
   가상 트레이드를 실행한다.
2. 아래 버킷별로 win_probability / avg_win / avg_loss를 집계한다.

   버킷 키: (direction, pressure_aligned)

   버킷 최소 표본: 30 trades
   미달 시 fallback 순서: (direction) → (train 전체)
3. 집계 결과를 저장한다.
   reports/di_kalman_mw/{symbol}_{timeframe}_train_stats.json
```

### Pass 2: 통계 적용

```text
1. Pass 1의 고정 통계만으로 validation/test 신호의
   raw_expected_value / pressure_adjusted_expected_value를 계산한다.
2. validation/test 구간에서 통계를 갱신하지 않는다.
3. walk-forward 모드에서는 window가 이동할 때마다
   해당 window의 train 부분으로 Pass 1을 다시 수행한다.
```

주의:

```text
[ ] train 구간 자체의 EV 필터 성능은 in-sample이므로
    리포트에 in-sample임을 명시한다.
[ ] costs는 §12의 fee + slippage 기본값과 동일한 값을 사용한다.
[ ] avg_win / avg_loss는 양수 크기(absolute magnitude)로 정의한다.
```

## 8. 진입 조건 토의안

### Variant A: P4 Prediction Entry

4번째 극점이 확정된 시점에서 진입한다.

Long 조건 초안:

```text
[ ] +DI pattern = W
[ ] -DI pattern = M
[ ] predicted_direction = long
[ ] pressure_adjusted_expected_value > 0
[ ] long_pressure_score >= 0.50
[ ] close > long_ma_filter 또는 ma_filter disabled
[ ] volatility filter 통과
[ ] 다음 봉 open 또는 signal bar close에 진입
```

Short 조건 초안:

```text
[ ] +DI pattern = M
[ ] -DI pattern = W
[ ] predicted_direction = short
[ ] pressure_adjusted_expected_value > 0
[ ] short_pressure_score >= 0.50
[ ] close < short_ma_filter 또는 ma_filter disabled
[ ] volatility filter 통과
[ ] 다음 봉 open 또는 signal bar close에 진입
```

장점:

```text
- 진입이 빠르다.
- P5가 나오기 전 움직임을 잡을 수 있다.
```

단점:

```text
- P5 예측 실패 시 손실 가능성이 크다.
- 신호 신뢰도가 낮을 수 있다.
```

---

### Variant B: P5 Confirmation Entry

5번째 극점이 실제로 확인된 뒤 진입한다.

Long 조건 초안:

```text
[ ] +DI W 이후 P5가 higher low로 확인
[ ] -DI M 이후 P5가 lower high로 확인
[ ] predicted_direction = long
[ ] pressure_adjusted_expected_value > 0
[ ] long_pressure_score >= 0.50
[ ] 다음 봉 open에 진입
```

Short 조건 초안:

```text
[ ] +DI M 이후 P5가 lower high로 확인
[ ] -DI W 이후 P5가 higher low로 확인
[ ] predicted_direction = short
[ ] pressure_adjusted_expected_value > 0
[ ] short_pressure_score >= 0.50
[ ] 다음 봉 open에 진입
```

주의:

```text
P5 극점은 실제 극점 발생 bar가 아니라,
reversal threshold를 충족한 이후에야 확정된다.

따라서 백테스트에서는 극점 idx가 아니라
confirmation_idx 이후에 진입해야 한다.
```

장점:

```text
- P4 예측 진입보다 신뢰도가 높다.
- 잘못된 M/W setup을 걸러낼 수 있다.
```

단점:

```text
- 진입이 늦다.
- 수익 구간 일부를 놓칠 수 있다.
```

## 9. 손절 조건 토의안

초기에는 아래 3가지를 모두 실험 가능하게 만든다.

### Stop Type 1: ATR Stop

Long:

```text
stop_price = entry_price - atr * atr_stop_mult
```

Short:

```text
stop_price = entry_price + atr * atr_stop_mult
```

기본값 후보:

```text
atr_len = 14
atr_stop_mult = 2.0
```

장점:

```text
- 구현이 단순하다.
- 변동성에 따라 손절폭이 조절된다.
```

---

### Stop Type 2: Price Swing Stop

Long:

```text
stop_price = recent_swing_low - atr * buffer_mult
```

Short:

```text
stop_price = recent_swing_high + atr * buffer_mult
```

장점:

```text
- 시장 구조 기반 손절이다.
```

단점:

```text
- swing 정의가 불안정하면 결과가 흔들릴 수 있다.
```

---

### Stop Type 3: DI Invalidation Stop

Long 무효화:

```text
+DI pressure가 약화되고
-DI pressure가 다시 우세해지면 청산
```

예시:

```text
long_pressure_score < 0.45
또는
+DI Kalman M 발생
또는
-DI Kalman W 발생
```

Short 무효화:

```text
short_pressure_score < 0.45
또는
+DI Kalman W 발생
또는
-DI Kalman M 발생
```

이 조건은 단독 손절보다 보조 청산 조건으로 사용하는 것이 좋다.

---

## 10. 익절 조건 토의안

### Take Profit Type 1: Fixed R Multiple

손절폭을 기준으로 익절한다.

Long:

```text
risk = entry_price - stop_price
take_profit = entry_price + risk * rr_target
```

Short:

```text
risk = stop_price - entry_price
take_profit = entry_price - risk * rr_target
```

기본값 후보:

```text
rr_target = 1.5
rr_target = 2.0
rr_target = 3.0
```

---

### Take Profit Type 2: Pressure-adjusted RR

DI pressure score에 따라 목표 손익비를 조절한다.

```python
base_rr = 1.5
rr_target = base_rr * pressure_rr_factor
rr_target = clip(rr_target, 1.0, 3.0)
```

예시:

```text
+DI 평균 압력이 -DI보다 강하면 long의 rr_target 증가
-DI 평균 압력이 +DI보다 강하면 short의 rr_target 증가
압력 방향이 불리하면 rr_target 감소
```

이 방식이 이번 전략의 핵심과 가장 잘 맞는다.

---

### Take Profit Type 3: Opposite Pattern Exit

Long 청산:

```text
+DI M 발생
또는
-DI W 발생
또는
bearish setup 발생
```

Short 청산:

```text
+DI W 발생
또는
-DI M 발생
또는
bullish setup 발생
```

장점:

```text
- DI 압력 구조가 실제로 바뀔 때까지 포지션을 유지할 수 있다.
```

단점:

```text
- 청산이 늦어질 수 있다.
```

---

### Take Profit Type 4: Trailing Stop

가격이 유리하게 움직이면 stop을 따라올린다.

Long:

```text
trail_stop = highest_close_since_entry - atr * trail_mult
```

Short:

```text
trail_stop = lowest_close_since_entry + atr * trail_mult
```

기본값 후보:

```text
trail_mult = 2.0
```

---

## 11. Time Stop

패턴 신호는 시간이 지나면 효력이 약해질 수 있다.

따라서 최대 보유 기간을 둔다.

```text
max_hold_bars = 20
max_hold_bars = 48
max_hold_bars = 72
```

4H 기준:

```text
20 bars ≈ 3.3일
48 bars ≈ 8일
72 bars ≈ 12일
```

Time stop 조건:

```text
entry 이후 max_hold_bars가 지나면 시장가 청산
```

## 12A. 체결 및 동시 터치 규칙 (신규 보강)

intrabar 체결 판정 규칙을 고정해 백테스트 결과의 모호성을 제거한다.

```text
1. 진입은 signal bar 다음 봉 open에서 체결한다.
   slippage는 진입/청산 각각 불리한 방향으로 slippage_rate만큼 반영한다.

2. stop / take_profit 터치 판정은 봉의 high / low로 한다.
   long:  low <= stop_price          → stop 체결
          high >= take_profit_price  → TP 체결
   short: 반대 방향 동일 로직.

3. 같은 봉에서 stop과 TP가 모두 터치되면 stop이 먼저 체결된 것으로
   가정한다 (보수적 가정, 고정 규칙).

4. 진입 봉에서는 open 이후부터 stop/TP를 평가한다.
   open이 이미 stop을 지나 있으면(gap) stop_price가 아니라
   open 가격으로 체결한다 (gap-through fill). TP도 동일.

5. trailing stop은 봉 close에서 레벨을 갱신하고 다음 봉부터 intrabar
   stop으로 적용한다. opposite pattern exit / time stop은 봉 close 기준으로
   평가하고 다음 봉 open에 체결한다.

6. 같은 봉 내 우선순위: stop/TP (intrabar) > opposite exit > time stop.
```

## 14. 성능 평가 기준

전략 후보는 최소한 아래를 만족해야 한다.

```text
[ ] train에서만 좋은 것이 아니라 validation에서도 유지

[ ] pressure-adjusted EV가 raw EV보다 선별력이 좋음

[ ] 비용 2배 가정에서도 완전히 붕괴하지 않음
```

초안 합격 기준:

```text

profit_factor > 1.2
max_drawdown 허용 범위 이내
num_trades >= 50

test 기준:
profit_factor > 1.0
expectancy > 0
특정 1~2회 거래에 수익이 몰리지 않을 것
```

신규 보강: num_trades 기준 적용 방식

```text
4극점 패턴은 희소 이벤트라 단일 symbol/timeframe의 validation에서
num_trades >= 50이 미달할 수 있다.
이 경우 동일 config로 BTC/ETH/SOL × timeframe 결과를 풀링한
거래 수로 평가하는 것을 허용한다.
단, 풀링 여부와 풀링 구성은 리포트에 반드시 명시한다.
```
