# option_plan — 현물 + 옵션(매수/매도) 절충 전략 (러프 계획)

> 한 줄: 현물로 베타, 옵션 매도로 변동성 리스크 프리미엄(VRP) 알파, 옵션 매수로
> 테일 방어 — 이 셋의 균형을 IV/스큐 신호로 동적 조절하는 손익-분포 조형 전략.
> 동기: yoon 계열(가격/볼륨 프로파일)에서 추가 알파가 더 안 나옴 → 가격에 없는
> 새 정보축(옵션=미래 기대 분포 + 보험 수요)으로 이동.

## 0. 배경 / 왜 옵션인가
- yoon1b/yoon1f는 cycle_multiple 프로파일(과거 실현 분포)에서 짜낼 정보를 다 추출.
  측정법(POC/VA/HVN)·해상도·비중·노출 전부 base 못 넘음. 생존편향 빼면 알파≈0,
  방어(낙폭 축소)만 진짜.
- 옵션은 가격/거래량과 본질적으로 다른 정보:
    · 미래 기대 분포(옵션 가격에서 역산한 risk-neutral density)
    · 보험 수요/공포·탐욕(IV 스큐, 풋콜비율)
  → "다른 정보" 알파원에 정확히 해당. 추가 알파 탐색의 첫 실제 출발점.

## 1. 핵심 아이디어 (손익 "모양" 조형)
- 현물 롱 = 시장 베타(수익 원천).
- 풋 매수 = 하방 보험(낙폭 컷), 비용.
- 콜 매도 = 상방 일부 포기 + 그 프리미엄으로 풋 비용 상쇄.
- 기본 골격 = 칼라(collar): "상방 캡 / 하방 보호"의 밴드형 손익.
  yoon1f가 현금비중으로 *간접* 방어 → 칼라는 옵션으로 *직접·정밀* 방어.

## 2. 이론적 전제 (잊으면 함정)
- (A) 공짜 점심 없음: 옵션은 무차익(no-arbitrage) 공정가. 보험 사면 평균적으로
  보험료만큼 손해, 팔면 보험료만큼 이득. 조합 자체로 기대수익을 공짜로 늘리는
  알파는 없다 — 분포 재배치(평균 그대로, 모양만 변경).
- (B) 유일한 진짜 알파 = VRP(변동성 리스크 프리미엄): IV가 RV보다 평균적으로 높음
  (보험 과수요). SPY 실측 스냅샷서 풋 IV 41~55% vs 콜 IV 14% 스큐로 확인.
  → 옵션 매도가 장기 양의 기대수익. 단 가격프로파일엔 없던 새 축.
- (C) 대가 = 테일 리스크: 옵션 매도는 평소 꾸준히 벌다 폭락 한 번에 토함
  (2018 volmageddon, 2020-03). yoon "낙폭 통제" 철학과 정면 충돌.
  → 반드시 테일 헤지(깊은 풋 매수) 동반.

## 3. 전략 골격 후보
- 방어 정합형(권장): 칼라 + 테일 헤지.
    콜 매도(VRP 수취) → 그 일부로 근거리 풋 + 깊은 OTM 풋(절벽 차단) 매수.
    "VRP를 일부 먹되 폭락은 막음".
- 동적 조절 로직(= "절충점 찾기"):
    · VRP 신호(IV/RV 비율): 클 때 매도 비중↑, 작을 때↓.
    · 스큐 가파를 때 풋스프레드로 비대칭 수취.
    · 시장 레짐(SPY 200MA 등, 기존 yoon 필터 재사용)별 행사가·만기 조정.
- 파라미터(러프): 만기(30~45D 롤), 행사가(델타 기준 예: 콜 0.25Δ 매도 / 풋 0.10Δ 헤지),
  롤 주기, 포지션 사이징(현물 대비 옵션 명목), 비용(옵션 스프레드 큼).

## 4. 데이터 — 최대 장벽
- 백테스트엔 과거 옵션 체인 히스토리(수년치 일별 strike×IV×OI)가 필요.
- 현 가용:
    · yfinance 옵션 체인 = "지금 스냅샷"만, 과거 없음. (로컬서 동작 확인됨)
    · KIS = 국내 KOSPI200 옵션 현재 시세 O / 미국 주식옵션 거의 X / 과거 히스토리 X.
    · 유료(CBOE/ORATS/OptionMetrics) = 과거 데이터 있으나 비용.
- 결론: 과거 데이터 없이는 백테스트 불가 → **스냅샷 적립으로 자체 히스토리 구축**이
  현실적 선행 단계(느리지만 무료). paper_trade 스냅샷 패턴 재사용.

## 5. 대상 시장 (결정 필요)
- 미국 SPY 옵션: yfinance 스냅샷 가능, 유동성·VRP 풍부. 단 KIS 미취급·과거데이터 없음.
- 국내 KOSPI200 옵션: KIS 현재 시세 O + 실거래 연계 가능. 과거는 적립 필요.
- 미정 → 둘 중 택1 후 수집기 설계. (실거래까지 보면 국내+KIS, 데이터 풍부함은 미국)

## 6. 단계별 로드맵 (러프)
1) 데이터 자산: 옵션 분포 스냅샷 적립기 (매일 체인 저장 → parquet, paper_trade 패턴).
   - strike/IV/OI/volume/spot/만기, 풋콜비율·ATM IV·스큐 요약.
2) 현황 분석 도구: RND(Breeden-Litzenberger: 콜가격 2차미분→미래 확률분포),
   IV 스큐·텀구조·VRP(IV vs 실현 RV) 시각화. 라이브 시장심리 모니터.
3) 페이퍼 시뮬: 적립된(또는 스냅샷) 데이터로 칼라+테일헤지 손익 곡선 모의.
   롤오버·비용 가정 명시. (정식 백테스트는 데이터 누적 후.)
4) 검증: VRP 수취가 비용·테일 후에도 양인지. yoon 방어와의 정합(낙폭) 점검.
5) 전략화: 충분히 검증되면 run_dashboard 계약(선언3+핸들러1)으로 편입 검토.
   단 옵션은 단일 OHLCV 계약과 안 맞아 별도 아티팩트/표현 설계 필요(오픈 이슈).

## 7. 기존 인프라 연결
- paper_trade.py 스냅샷 적립 패턴 → 옵션 스냅샷 수집기에 재사용.
- 시장 레짐 필터(SPY 200MA 등) yoon 엔진 로직 재사용 가능.
- KIS live 모듈(scripts/profile_sizing/live/kis_client.py, 미검증) → 국내 선택 시 실거래.
- run_dashboard 전략 계약(CLAUDE.md): 편입 시 forecast/trades/equity/metrics 매핑 필요
  (옵션 손익은 표준 trades 스키마와 거리 있음 → 매핑 설계가 숙제).

## 8. 리스크 / 오픈 이슈
- 과거 데이터 부재 = 검증 지연(스냅샷 적립은 수개월~). 최대 리스크.
- 옵션 매도 테일 = 헤지 없으면 방어 철학 배반. 헤지 비용이 VRP 갉아먹는 균형이 관건.
- 실행 마찰: 넓은 호가 스프레드·유동성·마진·롤 비용. 백테스트가 과대평가하기 쉬움.
- 대시보드 계약 적합성: 옵션 다리(leg) 구조를 기존 단일종목 trades에 어떻게 담나.
- 결론적 기대치: VRP는 진짜 알파 후보이나 "큰 수익"보다 "분포 조형 + 약한 초과수익".
  yoon과 같은 자세 — 화려한 알파가 아니라 정직한 엣지로 접근.
