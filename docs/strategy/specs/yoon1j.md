# 전략 명세: yoon1j (연속형 안전자산 슬리브)

> [[profile-portfolio-v1]] 엔진([[yoon1f]] 섹터 방어형)에 **연속형 안전자산
> 슬리브(safe-sleeve)** 를 더한 변형. "주식=공격 / 채권·금=방어 전담"으로 역할을
> 분리한다. 약세장에서 노출을 줄여 생긴 **현금 완충분(spare)** 을, 이진 전환이 아니라
> 방어 강도에 비례해 추세 ON 안전자산으로 **연속 회전**한다. 핸들러는
> `ProfilePortfolioHandler` 공유(전용 코드 없음, config + 엔진 오버레이만).

## 2. 핵심 아이디어 — 버퍼 회전(이진 전환이 아니라)

방어를 "노출 축소(현금화)"가 아니라 "포트폴리오 비율 조절(약세장 강세 자산으로
이동)"로 하자는 발상. 단, **이진 전환(공격↔안전 전액 스위치)은 실패한다** — 사전
검증에서 공격섹터↔안전자산 이진 로테이션은 MDD -31.5%로 현금 방어(-15.6%)에 패배했다
(신호 오류 시 전액이 잘못된 쪽에 실리는 휩쏘 + 안전자산 동반하락 노출).

yoon1j는 두 가지를 다르게 한다:

1. **버퍼만 회전** — 주식 북(섹터 공격)은 그대로 두고, 약세장에서 노출이 줄어 생긴
   *현금 완충분(spare=1−주식목표노출)* 의 일부만 안전자산으로 돌린다. 손실 상한이
   버퍼 크기로 제한된다(전액 전환이 아님).
2. **추세 게이트** — 각 안전자산은 *자기 추세가 ON(전봉 종가 > 자기 100MA)* 일 때만
   매수한다. 2022처럼 채권·금이 함께 빠지는 국면엔 사지 않고 현금으로 유지 →
   "약세장 강세 자산"의 조건을 신호로 강제. 이게 이진 전환과의 결정적 차이.

## 3. 엔진 동작 (`simulate_portfolio`, safe_sleeve 오버레이)

리밸런스(월간)마다, 주식 목표비중 산정 후:

```
spare       = max(0, 1 − Σ주식목표비중)
safe_spare  = max(0, spare − 숏헤지배정분)
safe_on     = {안전자산 s | 전봉 종가_s > MA(safe_ma_len)_s}
safe_budget = min(safe_max, safe_spare, safe_spare × safe_ratio)   (safe_on 비었으면 0)
              ↑ safe_spare 상한 = 무레버 보장(주식+안전+현금 ≤ 1)
각 s∈safe_on 목표비중 = safe_budget / |safe_on|     (균등 배분)
```

목표대로 안전자산 보유를 회전(거래비용 부과), 다음 리밸런스까지 보유. 레버리지·
공매도 없음(주식+안전+숏헤지+현금 ≤ 1). short_hedge와 동일 자리의 additive 오버레이로,
`safe_sleeve.enabled=false`(기본)면 기존 전략 동작 불변.

## 4. 유니버스·파라미터

| 항목 | 값 |
| --- | --- |
| 공격(주식) 유니버스 | XLK·XLF·XLV·XLE·XLI·XLP·XLY·XLU·XLB·XLRE·XLC (섹터 11) |
| 안전자산 슬리브 | TLT·GLD |
| safe_sleeve | `{enabled, symbols=[TLT,GLD], ratio=1.0, max=0.4, ma_len=100}` |
| top_k | 11 |
| 나머지 | yoon1f 상속(gain 1.0·SPY 100MA 필터·SMA 추세 오버레이·monthly) |

파라미터 스윕(ratio×cap×ma, 월간+거래비용 현실 모델)에서 **ma100이 ma150·ma200을
단조 우월**(추세 게이트가 빠를수록 안전자산 회복을 일찍 타고 일찍 빠짐), cap 0.3~0.5
구간이 위험조정 최적. r1.0/cap0.4/ma100 채택. (무레버 클램프 도입 후 ratio≥1.0은
safe_spare 상한에 걸려 동일 동작이므로 의미가 명확한 1.0으로 확정 — 이전 1.25는
완충분을 최대 8%p 초과하는 무비용 레버리지를 유발하던 버그였다.)

## 6. 판정

- **버퍼 회전 + 추세 게이트는 명확한 플러스**: 섹터 현금방어 대비 CAGR +2.1%p·
  Sharpe +0.18, **MDD는 동등~소폭 개선**(추세 게이트가 안전자산 동반하락을 회피).
  무레버 클램프 후에도 우위가 유지돼 공짜 레버리지 산물이 아님이 확인됐다.
- **vs SPY(buy&hold)**: 수익은 근소 양보(all 8.2% vs 8.7%, test 11.6% vs 14.8%)이나
  **낙폭·위험조정에서 압도** — all MDD -18.9% vs -55.2%·Sharpe 0.862 vs 0.528,
  test MDD -11.5% vs -24.5%·Sharpe 1.168 vs 0.903. 성격=방어형(시장 수익 거의 따라가며
  낙폭을 1/2~1/3로).
- vs yoon1f: 수익·Sharpe 우위, MDD는 양보(-18.9% vs -15.6%). yoon1f는 TLT/GLD를
  점수 유니버스에 넣어 방어에 깊이 관여시켜 MDD가 더 낮다. yoon1j는 역할 분리로
  공격 수익을 살리고 버퍼만 방어에 쓴다 → **위험조정·수익 우선이면 yoon1j, 최저
  낙폭 우선이면 yoon1f**.
- 한계/다음: 안전자산 2종(TLT·GLD)·동일가중 고정. 향후 ① 안전자산 추세강도 비례
  배분 ② 안전자산 유니버스 확장(IEF·SHY·금광주) ③ yoon1f처럼 TLT/GLD를 점수
  유니버스에도 동시 포함하는 하이브리드 비교가 오픈 이슈.
